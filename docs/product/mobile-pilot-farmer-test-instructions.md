# Mobile Pilot Farmer Test Instructions

Use these instructions when sharing an internal Android build with a small farmer test group.

## Goal Of The Test

We are testing whether quick voice notes and optional photos help capture farm information during real work.

This is not a production recordkeeping system yet.

## What To Try

1. Open the app.
2. Set up your farm name.
3. Add a few farm places, such as a field, bed, greenhouse, tunnel, wash/pack area, cooler, or storage area.
4. Record a farm note with your voice.
5. Add a photo when it helps explain what happened.
6. Review the note later in the farm-note timeline.
7. Try one manual harvest, material-use, or inventory-count record if relevant.
8. Create a media recovery package and save it somewhere you trust.

## What The App Does Not Do

- It does not transcribe your voice.
- It does not analyze photos.
- It does not upload your notes or photos.
- It does not sync across devices.
- It does not create an account.
- It does not back up to the cloud.
- It does not restore from a recovery file yet.

## What To Watch For

Please note:

- moments when recording a quick note felt useful;
- moments when typing would have been too slow;
- anything confusing about farm places;
- whether photos felt useful;
- whether the timeline helped you find notes;
- whether the recovery package wording felt clear;
- any point where you expected cloud backup, AI, or synchronization.

## Safety Reminder

Voice notes, photos, and records are saved on the device. Recovery files contain private farm information. Save or share them only where you are comfortable keeping private farm data.
