# Phase 1 Manual Smoke Test

Use a development build on a physical device or emulator.

1. Fresh install/open shows farm setup.
2. The app visibly communicates local/device-only storage and no-reception use.
3. Create a farm with a valid name.
4. Attempt an invalid blank farm name and observe clear validation.
5. Add one location.
6. Add one crop.
7. Add one material.
8. Add one countable item.
9. Close and relaunch the app.
10. Confirm all setup/reference data remains.
11. Enable airplane mode and confirm the Phase 1 flows still work.
12. Check touch targets, readability, color contrast, empty states, and validation messaging on a physical device.

Do not use this checklist to validate harvest, material-use, inventory-count, export, AI, server, sync, auth, sharing, or deployment behavior; those remain outside Phase 1.
